# Proof, Artifacts, and Traceability (Always On)

## Required proof for completion

Provide at least one:

- test output summary (command + result)
- build output (command + result)
- UI proof: screenshot/recording + steps to reproduce

## Change log

For any multi-file change, include:

- Files changed
- Why changed
- How verified

## If you cannot verify

State exactly:

- what you attempted
- what blocked verification
- the smallest next action to unblock

## IronClad business artifact tracking

- All business decisions must reference the SSOT Spec.
- Sales material changes must update the master README at `docs/ironclad/README.md`.
- New prospect data goes in `docs/ironclad/data/`.
- New sales/outreach materials go in `docs/ironclad/sales-marketing/`.
